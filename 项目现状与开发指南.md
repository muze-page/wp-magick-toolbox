# WP Magick Toolbox - 项目现状与开发指南

> **版本**: 2.0.84
> **定位**: 面向小站长的 WordPress 实用工具箱插件
> **代码库**: https://gitee.com/gitgreat/wp-magick-toolbox

---

## 一、 技术架构

### 1. 后端 (PHP)
- **版本**: PHP 7.0+
- **架构**: WordPress Plugin Boilerplate 变体
- **数据存储**: 所有设置存储在 `wp_options` 表中，键名为 `Magick_ToolBox_Option`，值为 JSON 对象。
- **核心类**: `MaBox_Admin` (位于 `admin/class-magick-mixtrue-admin.php`)
  - 负责注册菜单、加载资源、处理 AJAX 请求。
  - 提供 `get_config($config, $key, $default)` 方法用于读取配置（支持数组和对象）。
  - 提供 `get_seting($key)` 方法用于获取顶层配置。

### 2. 前端 (React)
- **技术栈**: React + TypeScript + Vite + Ant Design + TailwindCSS
- **构建目录**: `vite/admin/`
- **数据流**:
  - PHP 通过 `wp_localize_script` 将配置注入到 `window.dataLocal`。
  - React 通过 `DataContext` 读取配置。
  - 保存时通过 AJAX POST 到 `admin-ajax.php` (Action: `save_option_wmt`)。
- **类型定义**: `vite/admin/src/tool/interface.tsx`
- **默认值**: `vite/admin/src/tool/defaultVar.tsx`

### 3. 安全现状 (已完成加固)
- **SQL 注入**: 所有 SQL 查询已改用 `$wpdb->prepare()`。
- **CSRF**: 所有 AJAX 端点已添加 `check_ajax_referer` 验证。
- **XSS**: 所有动态输出已使用 `esc_html()` / `esc_url()` / `esc_attr()` 转义。
- **权限**: 敏感操作已添加 `current_user_can('manage_options')` 检查。

## 三、 开发规范 (如何添加新功能)

### 1. 后端 PHP 开发规范
- **位置**: `admin/partials/[category]/[feature_name].php`
- **结构**: 必须使用静态类，包含 `run($config)` 方法。
- **示例**:
  ```php
  if (!class_exists('Npcink_Page_New_Feature')) {
      class Npcink_Page_New_Feature {
          private static $option;
          public static function run($config) {
              self::$option = $config;
              // 注册 Hook
              add_action('wp_footer', array(__CLASS__, 'display'));
          }
          public static function display() {
              $val = MaBox_Admin::get_config(self::$option, 'new_feature_key');
              if ($val === true) {
                  echo 'Hello World';
              }
          }
      }
  }
  ```
- **加载**: 在对应的 `index.php` (如 `admin/partials/page/function/index.php`) 中添加 `require_once` 和 `::run()` 调用。
- **安全**: 
  - SQL 必须用 `$wpdb->prepare()`。
  - 输出必须用 `esc_html()` / `esc_url()`。
  - AJAX 必须验证 `check_ajax_referer` 和 `current_user_can`。

### 2. 前端 React 开发规范
- **类型定义**: 在 `vite/admin/src/tool/interface.tsx` 中添加字段。
  ```typescript
  export type PageFunction = {
      // ... existing fields
      new_feature_key: boolean; // 新增字段
  };
  ```
- **默认值**: 在 `vite/admin/src/tool/defaultVar.tsx` 中添加默认值。
  ```typescript
  const PageFunction = {
      // ... existing fields
      new_feature_key: false, // 默认关闭
  };
  ```
- **UI 组件**: 在对应的 Tab 组件 (如 `vite/admin/src/components/page/function.tsx`) 中添加 Ant Design 表单控件 (Switch, Input, Select 等)，绑定 `optionData.page.function.new_feature_key`。

### 3. 资源加载规范
- **CSS/JS**: 使用 `wp_enqueue_style` / `wp_enqueue_script`。
- **内联脚本**: 使用 `wp_localize_script` 传递 PHP 变量到 JS。
