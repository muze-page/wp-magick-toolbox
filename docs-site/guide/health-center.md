# 站点体检中心

站点体检中心是 v2.5 引入的核心功能，帮助你在后台首页一眼看清站点的配置健康度、潜在风险和优化建议。

## 体检评分

进入插件后台首页（仪表盘），最上方即为**体检中心**卡片。系统会基于以下维度实时计算综合评分（0-100）：

- **SEO 基础**：首页/文章/分类 TDK 是否配置
- **媒体优化**：是否开启图片 Alt 自动补全
- **站点优化**：是否移除 WP 版本号、隐藏顶部工具条、限制搜索频次
- **环境健康**：PHP 版本、WP 版本、伪静态、对象缓存、REST API 可用性
- **配置风险**：高风险模块、实验性模块和可能影响稳定性的配置

### 评分等级

| 分数 | 状态 | 颜色 | 含义 |
|------|------|------|------|
| ≥ 80 | 优秀 | 绿色 | 配置合理，运行安全 |
| 60-79 | 良好 | 黄色 | 基本合格，有优化空间 |
| < 60 | 需优化 | 红色 | 存在关键风险，建议立即处理 |

## 诊断项详情

体检中心会列出具体诊断：

1. **PHP 版本** — 低于 7.4 为 critical
2. **WordPress 版本** — 低于 6.0 为 warning
3. **固定链接（伪静态）** — 未启用为 warning
4. **对象缓存** — 未启用为 warning
5. **REST API 可用性** — 不可用为 critical
6. **已激活模块数** — 始终显示当前状态
7. **高风险模块检查** — 启用 high_risk/experimental 模块时 warning/critical
8. **SEO 基础配置** — 未配置时 warning
9. **媒体优化** — 未启用 Alt 补全时 warning

概览页另有“安全状态”面板。登录保护严格按国内生态登录安全模块当前真实的加载门槛（账号失败限制）展示；评论防护和信息暴露仍是相关开关摘要。该面板不等同于安全审计或防护效果证明。

## 风险与建议

- **关键风险**：红色数字显示当前检测到的 critical 项数量
- **建议优化**：黄色数字显示 warning 项数量
- **建议开启**：基于当前配置，系统推荐你开启的安全/SEO/优化功能
- **可一键优化**：对确定性高、低风险的配置项提供可预览的修复建议

## 一键优化建议

当体检中心识别到可自动修复的低风险配置时，会显示**可一键优化 N 项**按钮。点击后会打开预览窗口，列出每个建议的变更项、变更前后值和风险等级。

当前一键优化只覆盖确定性高的配置，例如移除 WP 版本号、限制搜索频次、图片 Alt 自动补全、首页 TDK 开关、隐藏顶部工具条和搜索日志。

::: warning 注意
点击**应用选中项**只会把建议应用到当前后台配置表单中，不会直接写入服务器。你仍然需要点击页面顶部或底部的**保存**按钮，确认保存后才会生效。
:::

## 联系技术支持

当体检中心检测到需要人工处理的问题时，会自动显示**联系技术支持**按钮。点击即可跳转到技术支持 Tab，你可以直接复制当前诊断报告用于沟通。

## 导出诊断报告

点击体检中心中的**导出诊断报告**按钮，系统会生成一份 Markdown 格式的报告，包含：

- 生成时间与体检评分
- 环境信息（PHP/WP/插件版本、固定链接、对象缓存、REST API）
- 体检项详情
- 风险模块/配置列表
- 可一键修复的建议
- 关键建议
- 需要人工处理的问题
- 搜索健康摘要（有搜索数据时）

报告优先复制到剪贴板，如果浏览器限制则自动下载为 `.md` 文件。

## 注意事项

- 体检数据**实时计算**，不存储任何持久化数据
- 每次刷新后台首页都会重新拉取最新诊断结果
- 一键优化建议不会自动保存配置，必须手动保存后才会写入数据库
