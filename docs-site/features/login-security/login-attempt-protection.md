# 登录尝试保护

登录尝试保护用于限制同一已存在用户、同一客户端 IP 在固定统计窗口内的重复失败尝试。达到阈值后，该组合会被临时阻止。

它不会建立纯账号全局锁，也不会建立纯 IP 全局锁，避免远程攻击者锁住整个账号或让共享出口 IP 下的所有用户一起失去登录能力。

## 配置项

| 设置 | 作用 |
|------|------|
| 开启登录尝试保护 | 启用或停用该能力 |
| 失败次数 | 统计窗口内允许的失败尝试数 |
| 统计窗口（分钟） | 聚合同一“用户 ID + 客户端 IP”失败尝试的时间范围 |
| 锁定时间（分钟） | 达到阈值后的临时阻止时长 |
| 可信代理 | 允许提供转发 IP 链的直接代理地址，每行一个精确 IP |

该能力对应当前服务端字段：

```text
attempt_limit_enabled
attempt_limit_count
attempt_window_minutes
lock_duration_minutes
trusted_proxies
```

## 可信代理

默认只信任服务器直接看到的 `REMOTE_ADDR`，不会直接相信浏览器或中间节点提交的 `X-Forwarded-For`。

只有当 `REMOTE_ADDR` 精确匹配“可信代理”列表时，运行时才会读取转发链。配置时请遵守以下规则：

- 每行只能填写一个有效 IPv4 或 IPv6 地址。
- 填写直接连接 WordPress 的代理地址，不要填写访客 IP。
- 不支持 CIDR、域名或通配符。
- 多级反向代理需要逐项列出所有可信跳点。
- 运行时从转发链右侧开始跳过可信跳点，第一个非可信 IP 才作为客户端地址。
- 转发链任一项目格式无效、全部是可信跳点或没有可识别客户端时，本次请求会跳过登录尝试限制。
- `REMOTE_ADDR` 不在可信列表时，运行时会完全忽略 `X-Forwarded-For`。

::: warning 错误配置风险
遗漏直接代理可能让多个访客被识别为同一个代理地址；把外部地址错误加入可信列表则可能让请求者影响来源判断。修改代理配置前，请确认服务器的真实网络拓扑，并保留一个已登录后台窗口。
:::

## 开启方法

1. 进入 **魔法工具箱** → **国内生态** → **登录安全**。
2. 开启“登录尝试保护”并确认风险提示；此时只是待保存更改，不会立即影响登录。
3. 在保存前设置失败次数、统计窗口和锁定时间；如站点位于 CDN 或反向代理后，同时展开“高级：可信代理”填写并核对代理出口 IP。
4. 使用全局保存按钮检查变更摘要，确认范围后保存。
5. 使用独立测试账号和无痕窗口验证错误密码、临时阻止及恢复登录。

## 恢复

上线前应熟悉 [登录尝试保护紧急恢复](/guide/config-recovery#登录尝试保护紧急恢复)。紧急常量 `MABOX_DISABLE_LOGIN_PROTECTION` 只用于恢复，不应长期保留。

## 能力边界

- 不修改或隐藏 WordPress 原生登录地址。
- 不提供验证码、双因素认证、登录通知、登录日志或后台 IP 白名单。
- 不替代强密码、及时更新、主机防火墙或专用安全插件。
- “限制匿名作者枚举”是另一项独立能力，不依赖本开关。
