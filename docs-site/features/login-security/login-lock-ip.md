# 登录失败锁定 IP

::: tip 推荐
安全建议开启。防止暴力破解攻击。
:::

::: warning 当前依赖
登录安全运行时目前以“账号失败限制”为统一加载门槛。仅开启 IP 锁定不会单独加载该模块。
:::

## 功能说明

当登录失败次数超过阈值时，自动锁定来源 IP，在指定时间内禁止该 IP 再次尝试登录。

## 使用场景

- 防止暴力破解密码
- 阻止自动化登录攻击
- 保护管理员账户安全
- 减少服务器资源消耗（大量失败请求）

## 开启方法

1. 进入 **魔法工具箱** → **国内生态**
2. 开启「登录安全」
3. 配置失败锁定参数
4. 保存配置

## 配置选项

| 选项 | 说明 | 推荐值 |
|------|------|--------|
| 失败次数限制 | 达到此次数后锁定 | 5 次 |
| 锁定时长 | 锁定持续时间 | 30 分钟 |
| IP 锁定 | 多次失败后锁定 IP | 开启 |
| IP 失败次数 | IP 级别失败阈值 | 10 次 |
| IP 锁定时长 | IP 级别锁定时间 | 60 分钟 |

## 安全建议

- 失败次数建议设置为 3-5 次
- 锁定时长建议 30 分钟以上
- 配合「自定义登录入口」使用效果更佳
- 如使用 CDN，需确保获取真实 IP

## 注意事项

- 锁定期间会显示明确的锁定提示
- 管理员可通过数据库手动解锁
- 锁定记录会写入 WordPress 选项表
- 如忘记管理员密码，需通过 FTP/数据库重置
