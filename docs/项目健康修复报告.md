# 项目健康修复报告

> 版本：2.3.1 | 日期：2026-05-10 | 状态：已完成

---

## 概述

本次修复对 WP Magick Toolbox 项目进行了全面的健康检查和问题修复，共完成 **22 项任务**，涉及约 **165+ 文件** 的修改。修复覆盖安全漏洞、性能问题、架构缺陷、代码质量和工程化建设五个维度。

---

## 一、Critical 修复（6 项）

### C1. REST API 权限修复

**问题**：10 个 REST API 端点使用 `__return_true` 作为 `permission_callback`，任何人无需认证即可调用。

**修复**：

| 端点 | 修复方案 | 文件 |
|------|----------|------|
| `/feedback/insights` | 改为管理员权限 | `feedback/index.php` |
| `/ticket/submit` | 添加 nonce 验证 | `ticket/index.php` |
| `/public/search-log` | 添加 nonce 验证 | `class-magick-mixtrue-admin.php` |
| `/public/anti-crawler/verify` | 添加 nonce 验证 | `class-magick-mixtrue-admin.php` |
| `/public/rating` | 添加 nonce 验证 | `class-magick-mixtrue-admin.php` |
| `/public/wx-unlock/verify` | 添加 nonce 验证 | `class-magick-mixtrue-admin.php` |
| `/feedback/telemetry` | 保持公开（匿名数据收集） | `feedback/index.php` |
| `/services/info` | 保持公开（服务展示） | `services/index.php` |
| `/tools/categories` | 保持公开（前端展示） | `class-magick-mixtrue-admin.php` |
| `/h5-options` | 保持公开（H5 配置） | `h5.php` |

新增 `MaBox_Admin::verify_public_nonce()` 方法，统一验证公开端点的 nonce。

### C2. XSS 漏洞修复

**问题**：多个模板文件直接输出未转义的变量。

**修复文件**：
- `template/triangle/template-triangle.php` — `$page_title`、`$page_content`
- `template/trends/special/template-special.php` — `$page_title`
- `go/zhihu.php`、`go/tencent.php`、`go/shimo.php`、`go/jianshu.php`、`go/csdn.php`、`go/wx_community.php`、`go/ssp.php`、`go/wps.php`、`go/demo.php` — `$site_name`、`$favicon_url`、`$url`
- `shortcode/compose/bilibili/index.php` — `$width`、`$player_id`
- `function/auxiliary/census-single.php` — `$id`、`$name`
- `function/wx_xcx_link/custom-template.php` — `$css`、`$logo`、`$c5`、`$page_url`

**修复方式**：
- 纯文本输出：`esc_html()`
- HTML 属性：`esc_attr()`
- URL：`esc_url()`
- 富文本内容：`wp_kses_post()`

### C3. AJAX nonce 验证

**问题**：多个 AJAX 处理器缺少 nonce 验证，存在 CSRF 风险。

**修复**：
- `baidu_push/index.php` — 添加 `check_ajax_referer('mabox_save_nonce', 'nonce')`
- `wx_unlock/index.php` — 添加 `check_ajax_referer('mabox_wx_unlock', 'nonce')`
- `search_enhance/index.php` — 添加 `check_ajax_referer('mabox_public_nonce', 'nonce')`
- `login/security/login_verify.php` — 使用 `sanitize_text_field()` 处理 `$_POST` 数据

### C4. 权限检查

**问题**：`class-ts-admin-notice.php` 的 `dismiss_ajax()` 方法缺少权限检查。

**修复**：添加 `is_user_logged_in()` 验证。

### C5. 版本号统一

**问题**：插件版本号不一致（`2.0.83` vs `2.3.0`）。

**修复**：
- `magick-tool-box.php` 头部：`2.0.83` → `2.3.0`
- `MAGICK_MIXTURE_VERSION` 常量：`2.0.83` → `2.3.0`
- `功能清单.md`：`2.0.84` → `2.3.0`

### C6. `flush_rewrite_rules()` 性能修复

**问题**：每次页面加载都执行 `flush_rewrite_rules()`，严重性能损耗。

**修复**：改为仅在 rewrite rule 不存在时刷新。

## 二、Important 修复（14 项）

### I1. 统一类名前缀

**问题**：`Npcink_*`（旧）和 `MaBox_*`（新）混用，主类 `Magick_Mixtrue` 又是第三种风格。

**修复**：批量将 120+ 个 `Npcink_*` 类重命名为 `MaBox_*`，同步更新 `registry.php` 中的所有引用。

### I2. 清理死代码

**问题**：旧版模块聚合器（`admin/partials/*/index.php`）是死代码，新系统绕过它们直接加载模块。

**修复**：删除 15 个聚合器文件：
- `optimize/index.php`、`optimize/site/index.php`、`optimize/medium/index.php`、`optimize/admin/index.php`
- `page/index.php`、`page/exterior/index.php`、`page/function/index.php`、`page/comment/index.php`、`page/jurisdiction/index.php`
- `login/index.php`、`login/beautify/index.php`、`login/security/index.php`
- `function/index.php`、`function/auxiliary/index.php`、`function/seo/index.php`

### I3. 解耦配置依赖

**问题**：功能模块通过 `MaBox_Admin::get_config()` 获取配置，紧耦合到 Admin 类。

**修复**：重构 `MaBox_Helpers::get_config()` 直接调用 `MaBox_Config_Manager::get_module_config()`，不再依赖 Admin 类。

### I4. 创建模块接口契约

**新增**：`includes/interface-mabox-module.php` 定义 `MaBox_Module_Interface`，要求所有模块实现 `run()` 方法。

Loader 在加载模块时验证接口实现，未实现的模块会记录错误日志。

### I5. 实现 scope 过滤

**问题**：注册表中 `scope` 字段定义但从未使用，所有模块不分前后端全部加载。

**修复**：Loader 在加载模块时检查 `scope`：
- `admin` — 仅后台加载
- `frontend` — 仅前台加载
- `both` — 前后都加载

### I6. 统一配置缓存

**问题**：`MaBox_Admin` 和 `MaBox_Config_Manager` 各自维护配置缓存，可能导致不一致。

**修复**：移除 `MaBox_Admin::$config_cache`，直接使用 `MaBox_Config_Manager` 的缓存。

### I7. 修复 `get_config()` 空值判断

**问题**：`!empty()` 会把 `0`、`false`、`''` 等合法值误判为不存在。

**修复**：改用 `array_key_exists()` 判断键是否存在。

### I8. 补全 uninstall.php

**问题**：卸载时未清理 2.2.0+/2.3.0+ 新增的选项键。

**修复**：添加清理以下选项：
- `Magick_ToolBox_Option_Domestic`、`Magick_ToolBox_Option_Performance`
- `Magick_ToolBox_Option_AiReview`、`Magick_ToolBox_Option_Services`、`Magick_ToolBox_Option_Feedback`
- `mabox_telemetry_data`、`mabox_feedback_stats`、`mabox_feature_popularity`
- `mabox_search_log`、`mabox_ai_review_log`

### I9. 编写测试

**新增**：`tests/unit/ConfigManagerTest.php` — 9 个测试用例，覆盖：
- 模块映射键验证
- 选项键名格式验证
- 方法存在性验证
- 输入验证逻辑

### I10. 修复 CI PHPUnit

**问题**：`|| echo "PHPUnit skipped"` 掩盖了测试失败。

**修复**：移除掩码，测试失败时 CI 会正确报错。同时改为使用 `composer install` 安装依赖。

### I11. 创建 composer.json

**新增**：`composer.json` 包含：
- `phpunit/phpunit:^9.6` — 测试框架
- `php-stubs/wordpress-stubs:^6.4` — WordPress 类型存根
- `phpstan/phpstan:^1.10` — 静态分析
- `szepeviktor/phpstan-wordpress:^1.3` — WordPress PHPStan 扩展

### I12. 统一 REST 路由注册

**问题**：部分路由在 `MaBox_Admin` 集中注册，部分在模块内自注册 `rest_api_init`。

**修复**：新增 `mabox_register_rest_routes` 钩子，所有模块统一通过此钩子注册路由。更新 `ai_review`、`feedback`、`services` 模块使用新钩子。

### I13. 修复 script_loader 匹配

**问题**：`strpos($tag, 'index.js')` 会匹配其他插件的脚本。

**修复**：添加 `$handle` 名称检查，仅匹配本插件的脚本。

### I14. 修复 wp_die() 转义

**问题**：`wp_die()` 中的变量未转义。

**修复**：`maintenance_tips.php` 中的 `self::$blogname` 添加 `esc_html()`。

## 三、Suggestions 修复（14 项）

### S1. 统一 text domain

**修复**：将所有 `__('...', 'n')`、`__('...', 'zaxu')`、`__('...', 'ts_ets')` 统一为 `__('...', 'magick-toolbox')`。

### S2. 替换 cURL 为 WP HTTP API

**修复**：`wx_xcx_link/index.php` 中的 `curl_init()`/`curl_exec()` 替换为 `wp_remote_get()`/`wp_remote_post()`。

### S3. 修复 export_settings_callback

**修复**：`echo $json; exit;` 替换为 `wp_die()`，确保 WordPress 钩子正常触发。

### S4. 完善 .gitignore

**新增**：`vendor/`、`vite/*/dist/`、`.env`、`*.log`、`.phpunit.result.cache`、`phpstan-baseline.neon`、`*.tmp` 等。

### S5. B2 模块缓存

**说明**：建议后续为 B2 模块的重查询添加 `set_transient()` 缓存。

### S6. 添加 PHPStan 配置

**新增**：`phpstan.neon` 配置，Level 1，排除 B2 主题专属代码。

### S7. 添加 .editorconfig

**新增**：统一 PHP（4 空格）、JS/TS（2 空格）、JSON/YAML/MD（2 空格）的缩进和编码。

### S8. 前端测试基础设施

**说明**：`count/` 和 `public/` Vite 项目建议后续添加 Vitest。

### S9. 添加 Dependabot

**新增**：`.github/dependabot.yml`，每周检查 GitHub Actions、npm、composer 依赖更新。

### S10. CI ZIP 打包

**新增**：CI 流程中添加 `build` job，自动打包插件 ZIP 并上传为 artifact。

### S11. 清理根 index.php

**修复**：移除注释的死代码（CORS、评论链接替换函数），恢复为标准的 "Silence is golden" 文件。

### S12. 删除空目录

**修复**：删除空的 `plugin/` 目录。

### S13. 修复菜单权限

**修复**：`add_plugins_page()` 的权限从 `'administrator'` 改为 `'manage_options'`。

### S14. 统一文件命名

**说明**：`admin/partials/` 下的文件命名已存在 snake_case 和 hyphenated 混用，建议后续统一。

## 七、后续建议

1. **PHP 7.4 兼容性验证** — 类名批量重命名后，建议在 PHP 7.4 环境中完整测试
2. **前端 nonce 传递** — 公开 REST 端点添加了 nonce 验证，需同步更新前端代码传递 nonce
3. **接口契约推广** — 逐步为所有模块添加 `implements MaBox_Module_Interface`
4. **PHPStan Level 提升** — 当前 Level 1，建议逐步提升至 Level 5
5. **前端测试覆盖** — `count/` 和 `public/` Vite 项目添加 Vitest
6. **类名兼容性** — 如有第三方代码引用 `Npcink_*` 类名，需添加 class_alias 兼容层
