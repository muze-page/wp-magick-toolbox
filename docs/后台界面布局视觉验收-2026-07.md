# 后台界面布局视觉验收

> 日期：2026-07-22
>
> 范围：WordPress 后台外壳的左对齐、1200px 内容边界和透明外层背景
>
> 结果：通过

## 验收目标

1. 移除桌面端外层容器造成的重复水平边距；
2. 移除外壳自身的灰色背景，但保留 CSS 隔离和设计 token 作用域；
3. Header、侧栏和正文保持同一左边界，最大宽度统一为 1200px；
4. 保留正文 24px 垂直间距和窄屏 12px 安全边距；
5. 不改变导航、搜索、保存状态、标签、字体、颜色或行为。

## 实机测量

- 1600px 视口：Header 与正文均为 1200px，并使用相同左右坐标；
- 1280px 视口：内容随可用宽度收缩，无横向溢出；
- 782px 视口：保留 12px 安全边距，侧栏折叠，主内容无横向溢出；
- 外壳背景为透明，正文顶部间距保持 24px；
- 页面重载、搜索、保存按钮和当前媒体体检标签均保持正常；
- 浏览器控制台无警告或错误。

## 设计判断

左对齐比页面居中更适合 WordPress 管理后台：它与 WordPress 固定侧栏形成稳定的工作区起点，宽屏下也不会让导航和正文随视口漂移。1200px 上限避免长表单过度拉伸，透明外层背景则去掉了“卡片外再套一层卡片”的视觉噪音。

`mabox-shell` 仍然保留，因为它承担 CSS 隔离、响应式和 token 作用域；只移除该容器不必要的边距与背景，不删除结构本身。

## 结论

- P0：无；
- P1：无；
- P2：重复边距和外层背景已经消除；
- 最终状态：通过。
