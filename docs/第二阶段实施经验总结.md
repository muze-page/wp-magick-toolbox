# WP Magick Toolbox 第二阶段实施经验总结

> 记录第二阶段（产品体验升级）实施过程中的技术决策、踩坑经验与规范建议，供后续阶段参考。

## 一、项目架构认知

### 1.1 技术栈
- **前端**：React 18 + TypeScript + Vite + Ant Design 5.x
- **后端**：WordPress PHP（插件架构）
- **状态管理**：React Context（DataContext）传递 optionData
- **构建产物**：`vite/admin/dist/` 下的 JS/CSS，由 PHP 通过 `wp_enqueue_script` 加载

### 1.2 配置数据结构
```
optionData
├── optimize        // 优化模块
│   ├── site        // 站点优化
│   ├── medium      // 媒体优化
│   └── admin       // 后台优化
├── page            // 页面模块
│   ├── feature     // 外观特效
│   ├── function    // 页面功能
│   ├── comment     // 评论设置
│   └── jurisdiction// 权限控制
├── function        // 功能模块
│   ├── seo         // SEO
│   ├── auxiliary   // 辅助功能
│   ├── b2          // B2主题
│   └── ...
├── login           // 登录页
├── shortcode       // 短代码
├── template        // 页面模版
└── h5              // H5页面
```

**关键约定**：
- 开关字段值类型不统一：`boolean` 或 `string`（"false" 表示关闭，其他字符串表示开启）
- 判断开启状态：`typeof val === 'boolean' ? val : val !== 'false' && val !== ''`

## 二、TypeScript 踩坑与规范

### 2.1 Ant Design Form 类型不匹配
**问题**：`AntConfig.from.labelCol` 类型为 `number | Record<string, number>`，但 Ant Design 的 `Form` 组件期望 `ColSpanType | undefined`。

**影响范围**：所有使用 Form 的组件（约 20+ 个文件）。

**解决方案**：
```tsx
// 推荐：在 AntConfig 定义处修复类型
// 当前项目采用：使用时 as any 断言
<Form
  labelCol={{ span: fromConfig.labelCol as any }}
  wrapperCol={{ span: fromConfig.wrapperCol as any }}
/>
```

**后续建议**：修改 `tool/tool.tsx` 中 `AntFrom` 接口的定义，使用 Ant Design 官方类型：
```ts
import type { ColProps } from "antd";
interface AntFrom {
  from: {
    labelCol: ColProps["span"];
    wrapperCol: ColProps["span"];
    maxWidth: number;
  };
}
```

### 2.2 .ts vs .tsx 文件扩展名
**问题**：包含 JSX 语法的文件使用了 `.ts` 扩展名，导致 `tsc` 编译失败。

**案例**：`riskyFeature.ts` 中使用了 `<ExclamationCircleOutlined />` 和 JSX 片段，编译报错 `TS1005: '>' expected`。

**规范**：
- **任何包含 JSX 语法的文件必须使用 `.tsx` 扩展名**
- 导入时也要使用 `.tsx` 扩展名：`import { x } from "@/tool/riskyFeature.tsx"`

### 2.3 未使用变量/导入
**问题**：TypeScript 严格模式下，未使用的导入或变量会导致编译失败（`TS6133`）。

**常见场景**：
- 从 antd 导入的组件未使用（如 `Radio`, `Tooltip`）
- 从 @ant-design/icons 导入的图标未使用
- 声明了 state 但未使用其 setter

**规范**：
- 删除所有未使用的 import
- 如果变量仅用于副作用，使用下划线前缀或改为直接调用

### 2.4 Ant Design 组件属性差异
**问题**：`Tag` 组件在 Ant Design 5.x 中不支持 `size` 属性。

**错误代码**：
```tsx
<Tag size="small">...</Tag>  // ❌ TS2322: Property 'size' does not exist
```

**正确做法**：
```tsx
<Tag>...</Tag>  // ✅ 或使用 style 调整字体大小
```

## 三、状态管理与数据流

### 3.1 DataContext 使用模式
```tsx
const { optionData, updateOption } = useContext(DataContext);

// 更新单个字段
updateOption("page", "feature", newFormData);

// 注意：updateOption 是同步的，不会自动触发后端保存
// 保存需调用 Save 组件中的 postData() -> saceOption(optionData)
```

### 3.2 表单状态同步模式
每个功能模块的标准模式：
```tsx
const [formData, setFormData] = useState(publicData || {});

const onValuesChange = (changedValues: Partial<FieldType>) => {
  setFormData((prev) => ({ ...prev, ...changedValues }));
};

useEffect(() => {
  updateOption("module", "submodule", formData);
}, [formData]);
```

**注意**：不要直接在 `onValuesChange` 中调用 `updateOption`，会导致每次按键都触发全局 re-render。

### 3.3 跨组件通信
- **父子通信**：Props 传递回调函数（如 `onNavigate`）
- **全局状态**：通过 DataContext 传递 optionData
- **客户端持久化**：localStorage（快照、收藏、自定义方案、不再提示记录）

## 五、客户端持久化规范

### 5.1 localStorage 键名命名
统一使用 `mabox_` 前缀：
- `mabox_snapshots` - 配置快照
- `mabox_favorites` - 常用功能收藏
- `mabox_custom_presets` - 自定义配置方案
- `mabox_risky_dismissed` - 高风险功能不再提示记录

### 5.2 数据序列化
所有存储数据必须经过 `JSON.stringify`/`JSON.parse`，并包裹 try-catch：
```ts
function getData<T>(key: string, defaultValue: T): T {
  try {
    const stored = localStorage.getItem(key);
    return stored ? JSON.parse(stored) : defaultValue;
  } catch (e) {
    console.error(`读取 ${key} 失败`, e);
    return defaultValue;
  }
}
```

### 5.3 存储大小限制
localStorage 通常限制 5-10MB。配置快照包含完整的 optionData（约 50-100KB），保留 5 个快照约 500KB，在安全范围内。

## 六、构建与部署

### 6.1 构建命令
```bash
cd vite/admin
npm run build  # 执行 tsc && vite build
```

### 6.2 构建产物
- `dist/index.js` - 主入口
- `dist/index.css` - 样式
- `dist/tab-*.js` - 懒加载的 Tab 组件

### 6.3 常见构建错误
| 错误 | 原因 | 解决 |
|------|------|------|
| TS1005: '>' expected | .ts 文件包含 JSX | 改为 .tsx |
| TS6133: 'X' is declared but never read | 未使用的导入/变量 | 删除未使用的代码 |
| TS2322: Type 'number \| Record' is not assignable | AntConfig 类型不匹配 | 使用 `as any` 或修复类型定义 |
| chunk size warning | 单个 chunk 超过 500KB | 正常现象，dashboard 包含大量预设数据 |

## 七、性能优化建议

### 7.1 已完成
- 懒加载各 Tab 组件（`React.lazy` + `Suspense`）
- 仪表盘使用 `useMemo` 避免重复计算
- 搜索索引静态定义，避免运行时构建

### 7.2 待优化
- **dashboard chunk 过大**（1.3MB）：将预设配置数据提取为独立 JSON 文件，动态导入
- **Form 全量 re-render**：每个表单组件独立管理状态，但更新时触发整个 Tab re-render
- **搜索索引维护**：当前硬编码 58 条，新增功能时需手动同步，建议从配置数据自动生成

## 八、代码规范建议

### 8.1 新增功能清单
实施新功能时，需检查以下清单：
- [ ] 是否在 `interface.tsx` 中定义类型
- [ ] 是否在 `defaultVar.tsx` 中设置默认值
- [ ] 是否在 `feature-search.tsx` 的 `searchIndex` 中添加搜索条目
- [ ] 是否为高风险功能（添加 `RISKY_FIELDS` 映射）
- [ ] 是否按需加载资源（PHP 端 `run()` 方法中检查开关）
- [ ] 是否支持移动端（表单在小屏幕下可正常使用）

### 8.2 Git 提交规范
建议按模块提交：
```
feat(dashboard): 添加站点健康评分
feat(presets): 添加企业官网推荐配置
fix(mobile): 优化平板横屏布局
```
