# 第三阶段实施总结

> **实施日期**: 2026-05-08
> **对应路线图**: 第三阶段「国内生态刚需功能」（第 7-12 周）
> **版本目标**: 2.1.0 → 2.2.0+
> **实施状态**: ✅ 全部完成

## 一、实施概览

第三阶段共实施 **10 个子任务**，新增 **2 个 Tab**（国内生态 / 性能优化）、**10 个 PHP 后端模块**、**13 个 React 前端组件**，覆盖路线图 3.1-3.10 全部内容。

**核心原则遵循**:
- 只做中国生态刚需，避免与海外插件直接竞争
- 延续第一阶段架构（配置拆分 + 模块注册表 + 按需加载）
- 所有新增功能通过注册表统一管理，支持风险标签
- 保持向后兼容，不破坏老用户配置

## 四、架构集成说明

### 4.1 配置存储

沿用第一阶段的配置拆分架构：

```
Magick_ToolBox_Option_Domestic    → domestic.compliance / baidu_push / wechat / comment_security / login_security
Magick_ToolBox_Option_Performance → performance.oss / seo_checker / media_health / search_enhance / db_clean
```

新增的两个 Option 键已注册到 `MaBox_Config_Manager::$module_map`，完整配置读取/保存逻辑无需改动。

### 4.2 模块加载

所有新增模块通过 `admin/modules/registry.php` 注册，沿用统一格式：

```php
'domestic.compliance' => array(
    'class'       => 'Npcink_Domestic_Compliance',
    'file'        => 'domestic/compliance/index.php',
    'option_key'  => 'domestic.compliance.icp_enabled',
    'category'    => 'domestic',
    'scope'       => 'frontend',
    'config_path' => 'domestic.compliance',
    'risk_tags'   => array('推荐'),
),
```

`MaBox_Module_Loader` 自动处理：
- 开关状态检测（`option_key` 为 truthy 才加载）
- 配置传递（`config_path` 读取嵌套配置传入 `run($config)`）
- 按需加载（未启用不注册 Hook，不占用内存）

### 4.3 前端资源

后台 React 应用：
- 新增两个懒加载 Tab 组件，通过 `React.lazy()` 按需加载
- 新增组件均使用 `DataContext` 消费配置，遵循现有 `updateOption(father, son, value)` 模式
- 表单使用 Ant Design 组件，布局与现有模块一致

前台 PHP 模块：
- 所有前台功能在 `run()` 方法内检查开关后再注册 Hook
- 资源（JS/CSS）通过 `wp_add_inline_script` / `wp_add_inline_style` 注入，避免额外 HTTP 请求

### 4.4 AJAX 端点

新增后台 AJAX 端点（无需额外注册，使用 `wp_ajax_` 前缀）：

| 端点 | 所属模块 | 功能 |
|---|---|---|
| `mabox_baidu_batch_push` | 百度推送 | 批量推送历史文章 |

| `mabox_seo_fix_alt` | SEO 检查 | 批量补全图片 Alt |

## 六、注意事项

### 6.1 百度推送 API

- 使用百度搜索资源平台的 `data.zz.baidu.com/urls` 接口
- **注意**: 2024 年后百度搜索资源平台 API 有调整，如遇到推送失败请确认最新接口地址
- 需要配置 `site`（完整域名，如 `https://www.example.com`）和 `token`（搜索资源平台提供）

### 6.2 微信 JSSDK

- 需要配置有效的 `appid` 和 `appsecret`
- 签名使用 7000 秒缓存的 `jsapi_ticket`，通过 `access_token` 获取
- 分享功能仅在微信内置浏览器中生效

### 6.3 对象存储

- 使用**原生 curl** 手写签名上传，不依赖 heavy PHP SDK，保持插件体积轻量
- 上传成功后如需替换历史图片域名，建议额外开发批量替换脚本（当前已实现运行时 URL 替换）
- **安全提醒**: Secret Key 存储在 WordPress Option 中，请确保数据库安全

### 6.4 登录安全

- 自定义登录地址修改后，原 `wp-login.php` 会自动重定向到新地址
- IP 白名单功能会拦截**所有非白名单 IP 访问后台**（包括已登录用户），请确保将自己的 IP 加入白名单后再启用

### 6.5 数据库清理

- 清理操作使用 `$wpdb->query()` 直接执行 DELETE，**不可恢复**
- 建议在清理前先导出数据库备份
- 自动清理任务通过 WP Cron 执行，需确保站点有正常访问流量以触发 Cron
