## 一、安全修复（P0 级）

### 1.1 文章统计功能不可用

**问题**: `wp_localize_script` 句柄不匹配，导致数据无法传递到前端 JS。

**修复**: `admin/partials/function/auxiliary/census-single.php:69`
```diff
- wp_localize_script(MAGICK_MIXTURE_NAME . '_index_js', 'dataLocal', $MaBox_array);
+ wp_localize_script(MAGICK_MIXTURE_NAME . '_census_js', 'dataLocal', $MaBox_array);
```

### 1.2 小程序接口缺失错误处理

**问题**: 微信 API 调用失败时静默失败，无错误提示。

**修复**: `admin/partials/function/wx_xcx_link/index.php`
- `wx_json_token()`: 添加 cURL 错误检查、JSON 解析验证、微信 API 错误码检查、access_token 存在性检查
- `get_link()`: 添加 cURL 错误检查、JSON 解析验证、微信 API 错误码检查、openlink 存在性检查
- `add_hello_header()`: 添加 token 错误传递检查
- `get_h5_options()`: 添加 WP_Error 到 REST Response 的转换

### 1.3 外链跳转 XSS 漏洞

**问题**: `$_GET['url']` 未经消毒直接使用。

**修复**: `admin/partials/page/function/go/index.php:4`
```diff
- $external_url = isset($_GET['url']) ? $_GET['url'] : '暂无';
+ $external_url = isset($_GET['url']) ? esc_url_raw(wp_unslash($_GET['url'])) : '暂无';
```

## 二、功能兼容性修复

### 2.1 全站变灰与表情冲突

**问题**: `filter: grayscale()` 应用于 `<html>` 元素，导致 OwO 表情选择器也变灰。

**修复**: `admin/partials/page/exterior/all_grey.php`
- 将 filter 从 `html` 移至 `body`
- 添加 `.OwO, .OwO *` 排除规则，保持表情色彩正常

### 2.2 隐藏分类时下载框泄露

**问题**: 仅通过 CSS `display: none` 隐藏下载框，HTML 仍发送到客户端。

**修复**: `admin/partials/page/jurisdiction/hide_category.php`
- 添加 `strip_download_boxes()` 方法，服务端移除下载框 HTML
- 支持 B2 主题下载框、通用下载框、短代码下载框三种模式

### 2.3 JS 文件头部加载改为底部

**问题**: 7 个前台 JS 文件在 `<head>` 中加载，阻塞页面渲染。

**修复**: 将以下文件的 `wp_enqueue_script` 第 5 个参数从 `false` 改为 `true`:
- `comment_emoji.php` (OwO.js)
- `scrolling/index.php` (平滑滚动)
- `share/index.php` (分享按钮)
- `merc_map/index.php` (足迹地图，2 个 JS)
- `b2/add_menu.php` (B2 商城统计)
- `census-single.php` (文章统计)

### 4.2 保留例外

以下查询因技术限制无法使用 `prepare()`:
- `SHOW TABLES` — MySQL 元命令
- `OPTIMIZE TABLE` — 表名作为标识符无法参数化
- `DROP TABLE` — 表名作为标识符无法参数化
- `information_schema` 查询 — 无用户输入

### 4.3 修复模式

```php
// Before
$result = $wpdb->get_var("SELECT COUNT(*) FROM {$wpdb->posts} WHERE post_type = 'revision'");

// After
$result = $wpdb->get_var($wpdb->prepare(
    "SELECT COUNT(*) FROM {$wpdb->posts} WHERE post_type = %s",
    'revision'
));
```

## 九、后续建议

### 9.1 待观察项

| 项目 | 说明 |
|------|------|
| SQL prepare() 性能 | 大量 prepare() 调用可能轻微影响性能，需监控 |

### 9.2 未来改进方向

| 优先级 | 项目 | 说明 |
|--------|------|------|
| 高 | 接口统一 | admin-ajax.php 与 REST API 混合使用，建议制定迁移策略 |
| 中 | 依赖管理 | 三个 vite 项目可考虑 monorepo 方案 (pnpm workspaces) |
| 中 | PHP 版本 | 当前要求 PHP 7.0+，建议升级至 PHP 8.0+ |
| 低 | 代码覆盖率 | 目标: PHP 核心模块 60%+, React 组件 40%+ |
